Recursion Summary

Recursion is a technique in which
a function calls itself
to solve a smaller part of a problem.

It is widely used in
Data Structures and Algorithms
for solving complex problems
in a simple and elegant way.

Important Concepts

Base Case
- The condition that stops recursion
- Prevents infinite function calls

Recursive Case
- The function calls itself
- Reduces the problem size

Call Stack
- Stores function calls during recursion
- Functions are removed when they finish execution

Advantages of Recursion

- Simplifies complex problems
- Code becomes shorter and cleaner
- Very useful for tree and divide-and-conquer problems

Disadvantages of Recursion

- Uses extra memory due to call stack
- May cause stack overflow if recursion is too deep
- Sometimes slower than iteration

Key Takeaways

- Every recursive function must have a base case
- Problem size must decrease in each call
- Recursion uses stack memory
- Many recursion problems can also be solved using loops

Recursion is a fundamental concept
that is widely used in
Tree, Graph, and Divide-and-Conquer algorithms.
